Manual de uso - Planilla 29 BIS

Guia simple para usar la planilla de pedidos, sin conocimientos tecnicos. Este manual explica que mirar, que tocar, que no tocar y como resolver las tareas mas comunes del dia a dia.

Cotizador + pedidos online Uso diario de gestion Actualizado: 04/05/2026

Para que sirve esta planilla

La planilla recibe los pedidos que entran desde la web de 29 BIS. A partir de ahi, sirve para:

  • Ver rapidamente cada pedido.
  • Revisar datos del cliente, archivos y observaciones.
  • Cambiar el estado del pago.
  • Cambiar el estado del pedido.
  • Archivar pedidos.
  • Eliminar pedidos viejos o de prueba.

Que pestañas usar

operacion

Es la pestaña principal. Aca se trabaja el dia a dia.

orders_archivo

Es el archivo/historial de pedidos que fueron archivados. Mantiene la misma logica visual de colores que la hoja operativa para que la lectura sea mas natural.

prices

Sirve para cambiar precios y disponibilidad de papeles.

dashboard

Es un resumen general. No hace falta tocarlo para operar pedidos.

No tocar: orders y meta. Son hojas internas del sistema. Si se editan manualmente, se puede desordenar el funcionamiento.

Pestaña principal: operacion

La pestaña operacion ya viene preparada para lectura clara:

  • Los pedidos nuevos se agregan automaticamente.
  • Las filas tienen color alternado suave para facilitar la lectura.
  • Los encabezados tienen colores para agrupar informacion.
Importante: las hojas operacion, orders y orders_archivo comparten la misma logica visual de encabezados para que sea facil reconocer cada bloque de informacion.
Grupo Columnas Para que sirven
Datos del pedido N° pedido, Fecha Identifican el pedido y el momento en que entro.
Datos del cliente Cliente, Telefono, DNI, Email Sirven para contacto y control del pedido.
Trabajo Tipo impresion, Tipo papel, Tamano, Faz Describen como se debe producir el trabajo.
Archivos Nombre archivos, Adjuntos, Observaciones Permiten abrir lo que subio el cliente y leer aclaraciones importantes.
Seguimiento Forma de pago, Estado pago, Estado pedido, Total, Fecha y hora de retiro, Urgente Sirven para gestionar la produccion y la entrega.

Como leer los colores de encabezado

  • Verde agua: identifica el pedido y la fecha.
  • Rosa: agrupa los datos del cliente.
  • Naranja: agrupa total, forma de pago y estados.
  • Negro: queda para los datos tecnicos del trabajo, archivos y detalles internos.

Que se actualiza solo

  • Cuando entra un pedido nuevo desde la web, la hoja operacion se actualiza sola.
  • Los colores y el orden de columnas se vuelven a aplicar automaticamente cuando la hoja se regenera.
En la practica: normalmente no hace falta tocar el boton Actualizar hoja operacion. Ese boton queda como respaldo por si alguna vez hace falta refrescar manualmente.

Que se hace todos los dias

  1. Entrar a la pestaña operacion.
  2. Buscar el pedido por numero o cliente.
  3. Hacer clic en Ver adjuntos para abrir la carpeta del pedido.
  4. Leer la columna Observaciones por si el cliente dejo instrucciones.
  5. Revisar y editar:
    • Forma de pago: muestra si el cliente eligio transferencia o pago en el local.
    • Estado pago: Pendiente o Pagado.
    • Estado pedido: En revisión, En preparación o Listo para retirar.

Menu 29 BIS Gestion

El menu fue simplificado para que queden solo opciones utiles y faciles de entender:

Opcion Para que sirve Cuando usarla
Abrir manual de uso Abre este manual desde el mismo menu. Cuando necesitan ayuda o recordar como se hace una tarea.
Actualizar hoja operacion Refresca la vista operativa. Si sienten que algo no se ve actualizado o quieren forzar una recarga.
Archivar por rango de filas... Mueve pedidos desde operacion a orders_archivo usando un rango de filas. Cuando el pedido ya no debe quedar en el operativo pero quieren conservarlo en historial.
Eliminar por rango de filas... Borra pedidos definitivamente desde operacion o orders_archivo y manda sus adjuntos a la papelera de Drive. Para limpiar pedidos viejos, de prueba o desfasados, tanto del operativo como del archivo.

Como archivar pedidos

  1. Entrar a operacion.
  2. Ir al menu 29 BIS Gestion.
  3. Elegir Archivar por rango de filas....
  4. Escribir el rango. Ejemplo: 4-4 o 4-10.
  5. Confirmar.
El pedido desaparece de operacion, pero sigue existiendo en orders_archivo.

Como eliminar pedidos

La forma mas estable es por rango de filas.

  1. Entrar a operacion.
  2. Mirar los numeros de fila de la izquierda.
  3. Ir al menu 29 BIS Gestion.
  4. Elegir Eliminar por rango de filas....
  5. Escribir el rango. Ejemplo: 4-23.
  6. Confirmar.
Importante: esta opcion elimina definitivamente los pedidos de orders. No los manda a archivo.
Conviene usar esta opcion para borrar pedidos viejos, pruebas o filas desfasadas que no interesa conservar.
Al eliminar desde operacion, tambien se mandan a la papelera de Google Drive los adjuntos de ese pedido.

Como eliminar lo que ya esta archivado

Lo archivado tambien se puede eliminar desde el menu, sin tener que borrar filas manualmente.

  1. Ir a orders_archivo.
  2. Mirar los numeros de fila de la izquierda.
  3. Ir al menu 29 BIS Gestion.
  4. Elegir Eliminar por rango de filas....
  5. Escribir el rango. Ejemplo: 2-5.
  6. Confirmar.
No borrar la fila 1. Esa fila es el encabezado.
Cuando eliminas desde orders_archivo, tambien se mandan a la papelera de Drive los adjuntos asociados a ese pedido.

Como cambiar precios

  1. Ir a la pestaña prices.
  2. Buscar la fila correspondiente al papel, tamaño, tipo de impresion y faz.
  3. Cambiar el valor de precio unitario.
Ese cambio impacta en el cotizador online.

Como ocultar un papel sin stock

  1. Ir a prices.
  2. Buscar el papel correspondiente.
  3. En la columna disponible, poner FALSE.

Cuando vuelva a estar disponible, volver a TRUE.

Que hacer si algo no se ve bien

  • Si un pedido no aparece enseguida: esperar unos segundos y, si hace falta, usar Actualizar hoja operacion.
  • Si falta una columna: revisar si esta oculta y, si hace falta, volver a regenerar la hoja.
  • Si un estado aparece como invalido: revisar que el valor coincida con el dropdown exacto.
  • Si una accion del menu no hace lo esperado: no probar muchas veces seguidas. Cortar y revisar antes de seguir.

Cosas que no conviene hacer

  • No editar manualmente la pestaña orders.
  • No borrar encabezados.
  • No mover columnas de operacion si no es necesario.
  • No borrar filas al azar en hojas internas sin saber de donde vienen.

Resumen rapido

  • Para trabajar pedidos: usar operacion.
  • Para guardar pedidos viejos: usar Archivar por rango de filas....
  • Para borrar pedidos de prueba o desfasados: usar Eliminar por rango de filas....
  • Para borrar archivo viejo: ir a orders_archivo y usar Eliminar por rango de filas....
  • Para cambiar precios: usar prices.
